# Project Agent Rules

## Commit Tracking

- From now on, commit every intentional project change, even small edits, so the
  project history stays traceable.
- Keep commits focused and named with clear conventional commit messages.
- Do not commit local secrets, `.env*`, runtime databases, generated cache, or
  one-off audit artifacts just to make the tree clean.
- If a change is intentionally left uncommitted, explain why before handoff.
